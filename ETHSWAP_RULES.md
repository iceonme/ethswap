# ETHSWAP V9.3-Innovation 策略运行规则详解

本文档整理了当前系统（V9.3-Innovation-v1.1）的核心运行逻辑，供日常维护和参数调整参考。

## 一、 网格计算规则 (3实体 + 2虚拟层)

### 1. 数据采集与采样 (分段逻辑)
- **初始/0点数据**：使用最近 **6 小时**（360 根 1 分钟线）。
- **日内重置数据**：使用最近 **4 小时**（240 根 1 分钟线）。
- **5段采样**：将时间窗口（如 360 分钟）均分为 **5 段**，取每段内的 **1个最高价** 和 **1个最低价**，共得 5 高 5 低。

### 2. 去极值算法 (去针)
- **平滑处理**：将 5 个最高价/最低价排序，**剔除最大值和最小值**，取中间 3 个值的平均值。
- **网格顶 (grid_top)**：3 个高价平均值。
- **网格底 (grid_bottom)**：3 个低价平均值。
- **目的**：有效防止单根异常插针带偏网格线，使网格紧贴实体活跃区。

### 3. 五层空间布局 (6条线)
- **步长 (Step)**：`(grid_top - grid_bottom) / 3`
- **VH (极高线)**：`grid_top + Step` (突破观察边界)
- **P3 (实体顶)**：`grid_top`
- **P2 (中转界)**：`grid_top - Step`
- **P1 (分界线)**：`grid_bottom + Step`
- **P0 (实体底)**：`grid_bottom`
- **VL (极低线)**：`grid_bottom - Step` (突破观察边界)

## 二、 网格重置触发机制

1.  **配额恢复**：每日 **00:00 (本地时间)** 准时恢复每日重置次数为 0。
2.  **突破观察**：若价格突破 **VL** 或 **VH**，开启 **2 小时** 观察期。
3.  **执行重置**：
    - 若 2 小时后价格仍未回归网格，且当天尚未进行过重置，则触发重置。
    - 使用 **最近 4 小时** 数据重新计算网格。
    - **持仓处理**：重置时现有持仓**保留原样**。

## 三、 下单与入场机制 (RSI 详细逻辑)

### 1. 入场区域定位
- **Layer -1 (VL-P0)**：虚拟低层，需满足 RSI 严苛条件做多。
- **Layer 0 (P0-P1)**：实体低层，不看 RSI 触发做多信号。
- **Layer 2 (P2-P3)**：实体高层，不看 RSI 触发做空信号。
- **Layer 3 (P3-VH)**：虚拟高层，需满足 RSI 严苛条件做空。

### 2. 动态 RSI 阈值 (策略核心)
系统根据 **LSTM/趋势强度** 自动切换阈值：

| 状态 | 强趋势 (Trend >= 1) | 弱趋势/震荡 | 备注 |
| :--- | :--- | :--- | :--- |
| **做多 (实体层)** | RSI <= **40** | RSI <= **35** | P0-P1 区域直接触发 |
| **做空 (实体层)** | RSI >= **70** | RSI >= **75** | P2-P3 区域直接触发 |
| **平多止盈** | RSI >= **60** | RSI >= **65** | - |
| **平空止盈** | RSI <= **50** | RSI <= **45** | - |
| **虚拟层严苛度** | ±5 RSI | ±5 RSI | 极值区需更极端 RSI |

### 3. 无网格模式 (极值交易)
- 在 2 小时观察期内，若触发特定 RSI 极值（强趋势 25/75，弱趋势 28/80），允许进行 **一次性** 极值建仓，捕捉超跌反弹或超涨回撤。

### 4. 风控参数
- **杠杆比例**：1x-3x (基于 ATR 波动率自适应)。
- **交易冷却**：两次操作间隔 **60 秒**。
- **分仓单位**：`本金 / 3 / 当前价` 乘以实时杠杆。
