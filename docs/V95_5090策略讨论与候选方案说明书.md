## 二、当前策略逻辑概览（v93）

### 1. 网格结构
当前 v93 策略不是传统密集网格，而是：

- **3 层实体网格**
- **2 层虚拟缓冲层**

核心结构：
- `entity_grids`: 实体边界
- `virtual_grids`: 上下虚拟层

前端显示时，会组合为 6 条线（极高、P3、P2、P1、P0、极低）。

### 2. 当前交易骨架
目前主逻辑偏“区间均值回归 + RSI 条件确认”，但写法比较硬：

- 低层偏多
- 高层偏空
- 中层偏观察
- 越界时进入 `no_grid` 模式

### 3. 当前趋势模块
代码里有一个名为 `lstm_trend()` 的轻量趋势判断函数，但实际并不是真正的 LSTM 模型，而是：

- 取最近 60 个价格点
- 计算平均收益率（动量）
- 计算收益率标准差（波动率）
- 输出 `-1 / 0 / 1`

当前它参与：
- RSI 阈值切换
- 同向仓位放大
- 交易风格调节

## 三、当前策略已发现的主要问题

以下问题是在 2026-03-24 多轮讨论中逐渐明确的。

### 问题 1：层与方向绑定过死
当前 `_grid_trading()` 的核心问题是：

- 低层主要允许开多
- 高层主要允许开空
- 还叠加“上半区 / 下半区”限制

这会导致：
- 层变成了方向开关
- 合约逻辑不够灵活
- 同一层不能根据持仓状态做不同动作

### 问题 2：条件过于苛刻，机会丢失较多
现行版常见过滤链条是：

- 到某层
- 还要到半层
- RSI 还要满足较极端阈值
- 还要当前没有某方向仓位

结果是：
- 很多看起来已经该做的点不做
- 信号稀疏
- 交易回合偏少

### 问题 3：趋势模块权力过大
当前趋势判断不仅影响仓位，还影响：
- RSI 阈值
- 行为节奏
- 进攻方向倾向

这会带来一个典型别扭感：

> 系统主骨架偏均值回归，但局部调制又想追趋势。

导致结果是：
- 主逻辑和调制逻辑容易冲突
- 看起来“复杂”，但未必更好

### 问题 4：空头侧明显不足
回测与运行观察都说明：
- 现行版空单机会偏少
- 很多高位放空机会由于条件过严没触发
- 导致交易分布不均衡

### 问题 5：展示与真实状态偶尔不同步
讨论中出现过：
- 截图里显示仍有 `0.6000 ETH` 持仓
- 但查询交易所接口时 `pos=0`

后续已确认：
- **截图是滞后显示**
- 实际上已平仓

这说明：
- Dashboard 偶尔存在显示延迟 / 缓存残留
- 真实状态应优先以交易所查询 + 本地委托记录联合判断

## 四、对当前策略问题的判断

### 当前策略不是“完全错误”
而是更接近于：

> **能跑，但太紧；有逻辑，但机会抓得偏少。**

它的主要问题不是方向性完全错，而是：
- 条件设置偏死
- 自适应角色分工不清
- 机会过滤过度

### 核心判断
本轮讨论最终形成的判断是：

1. 当前版本问题真实存在
2. 但不应在手机端草率大改
3. 应先通过回测和纸面比较确认方向
4. 若后续真的改，应采用“**最小改进版**”路线，而非重构式大改

## 五、候选思路总纲（后续优化方向）

### 方向 1：理顺网格买卖条件
核心思想：

> **层决定交易区域，不再决定唯一方向。**

即：
- 低区：偏开多 / 平空
- 中区：偏观望 / 换手 / 减仓
- 高区：偏开空 / 平多

注意是“偏向”，而不是“只能”。

### 方向 2：弱化趋势模块
核心思想：

> **趋势模块不再决定是否持仓，而是在持仓决定后负责“加多少仓、减多少仓”。**

即：
- 主规则决定方向
- 趋势只调节仓位力度

### 方向 3：保持均值回归主骨架
核心思想：

> 系统主骨架仍然是均值回归，不在这一版里混入完整追趋势逻辑。

原因：
- 当前网格结构、本地展示、重置规则都天然属于均值回归框架
- 如果同时改成趋势主导，会把变量混在一起，不利于验证和归因

### 2. 候选版的设计目标
该候选版只做**最小改进**，不做大重构：

#### 改动点 A：理顺区域逻辑
- 低区（-1,0）：偏开多 / 平空
- 中区（1）：观望
- 高区（2,3）：偏开空 / 平多

#### 改动点 B：固定 RSI 阈值
- 不再用趋势模块动态改 RSI
- 保持阈值稳定：
  - oversold = 25
  - overbought = 85
  - exit oversold = 40
  - exit overbought = 70

#### 改动点 C：趋势模块只调仓位力度
趋势判断只决定：
- 顺势时略增仓
- 逆势时略减仓
- 不再主导是否开多 / 是否开空

### 3. 明确没动的部分
候选版**没有修改**：
- 当前线上运行文件 `eth_swap_v93.py`
- 启动入口
- Dashboard
- 重置规则
- 数据源与执行器

所以：
- 当前线上运行不受影响
- 候选版仅用于离线比较和后续准备
